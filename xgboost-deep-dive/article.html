Deep Dive into Gradient Boosting Machines: The Power of XGBoost

Author: Jyothi Koushik Date: April 14, 2024 Read Time: 8 min

TL;DR

XGBoost (eXtreme Gradient Boosting) is an optimized distributed gradient boosting library. It uses a second-order Taylor expansion and L1/L2 regularization to achieve state-of-the-art performance on tabular datasets.

Benchmark Design Summary: Comparisons were performed on identical hardware (Local Env) using scikit-learn v1.8.0 and XGBoost v3.2.0.

In the landscape of modern machine learning, few algorithms have garnered as much prestige as XGBoost (eXtreme Gradient Boosting). In this article, we explore the theoretical foundations and specific optimizations that make it unique.

The Theory: From Boosting to XGBoost

Boosting grows trees sequentially. Each new tree is designed to correct the errors made by the previous trees using gradient descent.

The prediction at step \( t \) is:

\[ \hat{y}_i^{(t)} = \hat{y}_i^{(t-1)} + f_t(x_i) \]

The objective function used by XGBoost includes a second-order Taylor expansion:

\[ \text{Obj}^{(t)} \approx \sum_{i=1}^n [L(y_i, \hat{y}_i^{(t-1)}) + g_i f_t(x_i) + \frac{1}{2} h_i f_t^2(x_i)] + \Omega(f_t) \]

Code Implementation: XGBoost with scikit-learn

XGBoost provides a seamless integration with the scikit-learn suite. Here is the core implementation used in our benchmark:

import xgboost as xgb

# Initialize the model
model = xgb.XGBClassifier(
    n_estimators=100,
    learning_rate=0.1,
    max_depth=5,
    eval_metric='logloss'
)

# Train and Predict
model.fit(X_train, y_train)
y_pred = model.predict(X_test)

Hands-on Experiment: XGBoost vs. Random Forest

To demonstrate the efficacy of XGBoost, we conducted an experiment on the Breast Cancer Wisconsin (Diagnostic) dataset. This dataset consists of digitized images of fine needle aspirates (FNA) of breast masses.

Dataset Breakdown

  • Total Instances: 569
  • Features: 30 numeric, predictive attributes
  • Class Distribution: 212 Malignant, 357 Benign
  • Train/Test Split: 80% / 20% (114 test samples)
  • Pre-processing: Features were loaded as raw numeric values without scaling to test the tree models' robustness.

While Random Forest achieved an accuracy of 0.9649, XGBoost followed closely with 0.9561.

This result highlights a key ML truth: on smaller tabular datasets, the simpler decision boundary of Random Forest can sometimes generalize better than the aggressive optimization of XGBoost.

Appendix: Benchmark Design

The benchmarking was conducted using the following environment:

  • Hardware: Windows 11 (x64)
  • Runtime: Python 3.13.0
